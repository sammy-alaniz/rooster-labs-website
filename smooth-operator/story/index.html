Smooth Operator

Why I built a native DJI mission planner for Mac and iPhone.

about story support privacy

Link to app: https://apps.apple.com/au/app/drone-smooth-operator/id6760623759

I started Smooth Operator because I wanted to plan DJI missions somewhere other than my phone. Smooth Operator is a native macOS and iOS mission planning application for DJI drones that use the DJI Fly app.

An ideal Smooth Operator workflow starts with your Mac. Using your mouse pointer, you can create waypoints, add points of interest, and verify your route with different map imagery and airspace information. After a user completes their mission, they can save the mission to iCloud. Saving the mission to iCloud enables the user to load the mission onto their iPhone using Smooth Operator iOS.

Smooth Operator iOS has almost feature parity with macOS, with plans to reach full parity in progress, and the user can verify any last details. Now the user can go to the database tab. On this tab, they can adjust for launch altitude and then save the mission to DJI Fly. No manual file manipulation.

Before Smooth Operator, you had a few options to create a mission. Those include Litchi, Maven, Dronelink, and Rainbow. Typically in their workflow, you could create a mission but would then have to manually change files on your iPhone for DJI Fly to recognize the new mission. DJI Fly is the only iOS app that can fly newer DJI consumer drones. Luckily, DJI Fly's app filesystem can be accessed by Smooth Operator, and by any other app.

DJI Fly mission plans are recorded in two forms. First, they are placed in a SQLite database for cataloging and metadata. But the full mission plans are saved as a KMZ file in a nearby directory. KMZ is the mission plan and the source of truth for the complete route.

The database is located at:

<dji-fly-dir>/wayline_mission/mission_db/wpmz.sqlite3

KMZ files are located at:

<dji-fly-dir>/wayline_mission/<missionId>/<missionId>.kmz

I do not know why there are two forms to save the same mission, but at a high level, the database holds the UUID of the KMZ file, its file path, and some of the data from the KMZ. The KMZ holds the complete picture of the mission.

It was interesting to see that a DJI mission is simply two zipped XML files. To add actions to the mission, you add XML blocks to an XML file with appropriate values. So in all, this is just a fancy XML manipulator. But a useful one.
